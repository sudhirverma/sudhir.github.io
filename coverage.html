Coverage tool in Python

coverage.py tool use for measuring the code coverage Python program. It check your program which part of the code have been executed.

The idea bhiend using the coverage tool is that it is easy way to become acquainted with python code, to increase the test coverage for python stdlib. we like to have 100% of coverage, but any increase is good one but getting 100% coverage it not always possible there could be platform-specific code that will not execute for you, error in the output.

For installation and checking the version

Installation of coverage.py

$ pip dnf install coverage

To check the version of coverage tool the step are follow

$ coverage --version

Coverage.py, version 4.0.3

$ python -m coverage --version

Coverage.py, version 4.0.3.

Coverage tool have number of command which determine the action of coverage tool
  • Run : It run the python program and collect the executable data
  • report : report command generate the coverage report
  • html : it generate the html listings with coverage result
  • xml : Produce an xml report with coverage result
  • annotate – Annotate source files with coverage results.
  • erase – Erase previously collected coverage data.
  • combine – Combine together a number of data files.
  • ebug – Get diagnostic information.
